可以看到这个box盒子上会有很多的属性。这些属性大部分就是和JavaScript盒模型相关的。接下来把相关的盒模型属性分类一下。

在将盒模型分类之前，要理清一个看似简单，其实会忽略的一个问题，那就是什么是内容的宽高？，什么又是真实内容的宽高？。

内容的宽度和高度：设置的width/height这两个样式，就是内容的宽和高，内容的高取决于是否设置了height，如果height设定了固定值（比如200px），那么即使在内容溢出的情况下，height的值仍然是设定的固定值（200px）。如果没有设置height固定值，那么height会根据内容自适应，这样获取的值就是真实内容的height，width同理。

真实内容的宽度和高度：这个代指的是实际内容的宽高（和设置height/width没有必然的联系）。例如设置height为200px，如果内容溢出了，那么真实内容的高度是要把溢出的部分算上的。

理清了上面两个概念以后，接下来就是要将盒模型的属性分一下类了。

client系列（当前元素的私有属性）

  • clientHeight/ clientWidth: 内容的宽度/高度 + 左右/上下padding（和内容溢出没有关系）
  • clientLeft: 左边框的宽度，clientTop: 上边框的高度（即borderWidth[left, top]的值）

offset系列（当前元素的私有属性）

  • offsetWidth/ offsetHeight: clientWidth/clientHeight + 左右/上下边框（和内容溢出也是没有任何关系）
  • offsetParent: 当前元素的父级参照物
  • offsetLeft/ offsetTop: 当前元素的外边框距离父级参照物的内边框的偏移量

scroll系列（当前元素的私有属性）

  • scrollWidth/ scrollHeight: 和clientWidth/ cilentHeight一模一样（前提是：容器中的内容没有溢出的情况下）

    如果容器的内容有溢出，那么就是如下情况：

  • scrollWidth: 真实内容的宽度（包括溢出）+ 左padding

  • scrollHeight: 真实内容的高度（包括溢出）+ 上padding

    获取到的结果，都是约等于的值，因为：同一个浏览器，是否设置overflow=hidden，对于最终的结果是有影响的，在不同的浏览器中获取到结果也是不相同的

  • scrollLeft/ scrollTop: 滚动条卷去的宽度/高度

关于JavaScript盒子模型取值的问题

通过这些个属性值获取的结果永远不可能出现小数，都是整数；浏览器在获取结果的时候，会在原来真实的结果上进行四舍五入。

关于操作浏览器本身的盒模型信息

这要涉及到操作浏览器本身的盒模型信息，不管要操作属性，不管使用的是什么浏览器，也不管是获取还是设置，想要都兼容的话，需要写两套。

clientWidth/ clientHeight是当前浏览器可视窗口（一屏幕）的宽度和高度。

scrollWidth/ scrollHeight 是当前页面的真实宽高（所有屏加起来的宽度和高度），其值是一个约等于的值。
